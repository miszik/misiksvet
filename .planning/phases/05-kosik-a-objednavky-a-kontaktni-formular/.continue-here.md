3 problémy k opravení (všechny v index.html + order.js):

**1. PPL widget** — inline zobrazení vypadá ošklivě + nefunguje správně
- Oprava: vrátit modal overlay (jako Balíkovna), ale s data-mode="default" uvnitř
- Modal musí používat CSS visibility:hidden (NE display:none / hidden attr) aby widget mohl inicializovat
- Přidat zpět openPplModal(), closePplModal() do order.js
- Přidat zpět "Vybrat PPL výdejní místo" button do #ppl-box-section
- Přidat zpět PPL modal + overlay HTML za Balíkovna modal v index.html
- CSS: .ppl-modal--hidden { visibility:hidden; pointer-events:none; z-index:-1 } (EXISTUJE v style.css)
- Modal HTML má mít: id="ppl-modal" class="pickup-modal ppl-modal--hidden"

**2. Validace emailu nefunguje** — žádná chyba když email nemá @ nebo doménu
- Přepsat validateForm() z browser reportValidity() na inline chybové texty pod polem
- Funkce showInputError(el, msg) / clearInputError(el) — vytvořit p.form-error dynamicky pod inputem
- Pro email: zkontrolovat el.value.includes('@') + el.checkValidity()
- Zavolat firstInvalid.focus() jen jednou (první chybné pole)

**3. Telefon je příliš přísný** — buzeruje uživatele
- Stačí jen required (pole není prázdné), žádná validace formátu
- Zkrátit error message na: 'Vyplňte prosím telefonní číslo.'

</remaining_work>
